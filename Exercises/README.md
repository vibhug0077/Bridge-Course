# Exercises

Use these exercises after reading each unit. Add your solutions in this folder or in notebooks.

## How to Use This Exercise Bank

Each unit should be practiced in three passes:

1. Foundation: checks basic vocabulary and direct application.
2. Core: requires a complete explanation, program, diagram, or calculation.
3. Challenge: combines topics and requires testing, interpretation, or design.

For programming tasks, submit:

- Problem statement.
- Input and output description.
- Algorithm or pseudocode.
- Source code.
- At least three test cases.
- Short note on errors found and fixed.

For theory tasks, submit diagrams, tables, and explanations in your own words.

## Unit 1: Computer Fundamentals

### Foundation

1. Draw an input-process-output-storage diagram for online food ordering.
2. Classify ten computer components as input, output, processing, memory, storage, or communication.
3. Define data, information, hardware, software, CPU, RAM, SSD, DNS, URL, and protocol.

### Core

1. Explain an ATM transaction as a complete computer system with hardware, software, data, network, and users.
2. Compare a laptop, server, smartphone, and embedded system using purpose, hardware, software, and constraints.
3. Trace how DNS helps a browser open a website.
4. Prepare a hardware selection table for programming, video editing, and office use.

### Challenge

1. Design a smart classroom system with devices, network, applications, data flow, and security risks.
2. Diagnose five computer performance problems using symptoms, possible causes, and first checks.
3. Compare public Wi-Fi and campus LAN using bandwidth, latency, reliability, and security.

## Unit 2: C Programming

### Foundation

1. Write an algorithm and flowchart for finding the largest of three numbers.
2. Write C programs for simple interest, area of circle, Celsius to Fahrenheit, and swapping two numbers.
3. Explain `printf`, `scanf`, `main`, header files, variables, constants, and format specifiers.

### Core

1. Write a C program to print a multiplication table.
2. Write a C program to store marks in an array and calculate sum, average, highest, and lowest.
3. Write a grade calculator using input validation and `else if`.
4. Create a menu-driven calculator using `switch`.
5. Count vowels in a C string.

### Challenge

1. Build a student result program using `struct Student` and arrays.
2. Read numbers until `-1` and compute count, sum, average, maximum, and minimum.
3. Print four patterns using nested loops and explain the role of row and column counters.
4. Debug five faulty C programs and classify errors as syntax, logic, or runtime.

## Unit 3: Python

### Foundation

1. Print your profile using variables and formatted output.
2. Convert input strings to `int` and `float` for calculations.
3. Create examples of `int`, `float`, `str`, `bool`, `None`, list, tuple, set, and dictionary.

### Core

1. Write a Python program to count vowels, consonants, digits, and spaces in a string.
2. Create a dictionary for a student and print each key-value pair.
3. Write a Python program to find maximum and minimum values in a list without using `max` or `min`.
4. Use sets to find common, unique, and combined course selections.
5. Build a menu-driven calculator using `match` or `if elif`.

### Challenge

1. Create a command-line student analytics system using lists and dictionaries.
2. Validate marks using a `while` loop until correct input is entered.
3. Read numbers until `0` and print count, sum, average, highest, and lowest.
4. Convert one C program from Unit 2 to Python and explain the difference in style.

2. Draw truth tables for AND, OR, and NOT.
3. Decide whether given mappings are functions or relations.
4. Calculate simple probabilities using coins, dice, and cards.

### Core

1. Draw a weighted graph for five cities connected by roads.
2. Convert a graph into adjacency list and adjacency matrix form.
3. Perform BFS and DFS manually on a small graph.
4. Find mean, median, mode, range, variance, and standard deviation for a dataset.
5. Choose appropriate charts for five data questions and justify each choice.

### Challenge

1. Model a university course registration system using sets, relations, and functions.
2. Simulate 10,000 coin tosses in Python and compare observed probability with theoretical probability.
3. Analyze a student marks dataset using statistics and probability.
4. Explain how graph theory, probability, and statistics appear in a recommendation system.

## Unit 6: Computer Architecture

### Foundation

1. Draw the fetch-decode-execute cycle.
2. Compare registers, cache, RAM, ROM, SSD, and HDD.
3. Define ALU, control unit, register, cache, interrupt, DMA, address bus, data bus, and control bus.

### Core

1. Trace how `sum = a + b` may execute using registers, ALU, and memory.
2. Explain polling and interrupts with keyboard and printer examples.
3. Draw address, data, and control bus roles in a memory read operation.
4. Calculate addressable locations for 8-bit, 16-bit, 20-bit, and 32-bit address buses.
5. Explain temporal and spatial locality using arrays.

### Challenge

1. Prepare a real system architecture report for your laptop or lab PC.
2. Identify bottlenecks for booting, compiling code, opening a large dataset, and video calling.
3. Draw an I/O path for reading a file from SSD into a running program.
4. Compare CPU-bound, memory-bound, disk-bound, and network-bound workloads.

## Final Capstone Exercise

Build a student performance analysis project that connects the full bridge course:

1. Use C or Python to store student data and calculate results.
2. Use mathematical ideas such as sets, relations, probability, and statistics to analyze the data.
3. Create charts in Excel, Python, or R.
