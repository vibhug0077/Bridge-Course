# Glossary

| Term | Definition |
| --- | --- |
| Algorithm | Step-by-step method for solving a problem. |
| ALU | CPU component that performs arithmetic and logical operations. |
| Application Software | Software built for user tasks such as writing, browsing, or coding. |
| Array | Collection of same-type values stored using indices. |
| Bus | Communication path between computer components. |
| Cache | Small, fast memory near the CPU for frequently used data. |
| Compiler | Program that translates source code into executable code. |
| Computer Architecture | Organization and behavior of computer system components. |
| Control Flow | Order in which program statements execute. |
| CPU | Central Processing Unit; executes instructions. |
| Data | Raw facts that can be processed. |
| Data Structure | Organized way to store and access data. |
| DNS | Domain Name System; translates domain names to IP addresses. |
| Function | Mapping from input to exactly one output; also a reusable code block. |
| Graph | Structure of vertices and edges. |
| Hardware | Physical components of a computer. |
| Information | Processed data with meaning. |
| Interpreter | Program that runs code directly, often line by line or statement by statement. |
| IP Address | Numeric network address for a device. |
| Kernel | Execution engine for a Jupyter notebook language. |
| Loop | Programming structure that repeats statements. |
| Memory | Storage used while programs run. |
| Notebook | Interactive document containing text, code cells, and outputs. |
| Operating System | System software that manages hardware and programs. |
| Probability | Measure of uncertainty or likelihood. |
| Program | Set of instructions executed by a computer. |
| Random Variable | Numeric value assigned to outcomes of a random experiment. |
| RAM | Volatile main memory used by running programs. |
| Relation | Connection between elements of sets. |
| Set | Collection of distinct elements. |
| Software | Programs and instructions used by a computer. |
| Statistics | Methods for collecting, summarizing, and interpreting data. |
| Variable | Named storage location or named value in a program. |
| Visualization | Graphical representation of data. |
| Address Bus | Bus that carries the address of the memory or I/O location being accessed. |
| Addressable Memory | Number of unique locations that can be referred to by an address system. |
| Bandwidth | Amount of data that can be transferred per unit time. |
| Binary | Base-2 number system used by digital computers. |
| Bottleneck | Slowest limiting part of a system or workflow. |
| Byte | Group of 8 bits. |
| Conditional Probability | Probability of an event when another event is already known. |
| Control Bus | Bus that carries control signals such as read, write, clock, and interrupt. |
| Data Bus | Bus that carries actual data values between components. |
| Debugging | Process of finding, understanding, and fixing errors. |
| Device Driver | Software that allows the operating system to communicate with hardware. |
| DMA | Direct Memory Access; allows devices to transfer data to memory with limited CPU involvement. |
| Domain | Set of valid inputs for a function. |
| Expected Value | Long-run average value of a random variable. |
| Flowchart | Diagrammatic representation of an algorithm. |
| Format Specifier | Placeholder used by input/output functions to interpret data types. |
| Function Range | Set of outputs actually produced by a function. |
| Histogram | Chart showing the distribution of numeric values using bins. |
| IDE | Integrated Development Environment for writing, running, and debugging code. |
| Interrupt | Signal from hardware or software requesting CPU attention. |
| Latency | Delay before data transfer or response begins. |
| Locality | Tendency of programs to reuse recently accessed or nearby data. |
| Mutable | Describes data that can be changed in place. |
| Pseudocode | Informal structured description of an algorithm. |
| Register | Very small, very fast storage inside the CPU. |
| Scope | Region of a program where a variable or name can be used. |
| Sentinel Value | Special input value used to stop repeated input. |
| Short-Circuit Evaluation | Evaluation strategy where logical expressions stop once the result is known. |
| Standard Deviation | Measure of typical distance from the mean. |
| Truth Table | Table listing logical expression results for all input combinations. |
| Virtual Memory | Operating system technique that gives processes an abstract memory space. |
| Volatile Memory | Memory that loses contents when power is removed. |
